# Contributing

Thank you for your interest in contributing to this project. All contributions are welcome,
whether they are bug reports, feature suggestions, documentation improvements, or code changes.

## Ways to Contribute

- Report bugs via [issue templates](https://github.com/NWarila/Secure-RockyLinux9-Template/issues/new/choose)
- Suggest improvements or new features
- Submit pull requests
- Improve documentation or examples

## Getting Started

1. Fork the repository and create a feature branch from `main`.
2. Make your changes.
3. Run `pre-commit run --all-files` to verify all checks pass.
4. Open a pull request against `main`.

## Commit Messages

This project enforces [Conventional Commits](https://www.conventionalcommits.org/).
All commit messages must follow this format:

```text
<type>: <description>

[optional body]

[optional footer(s)]
```

Allowed types: `feat`, `fix`, `ci`, `docs`, `refactor`, `test`, `chore`, `security`

## Reporting Bugs

When reporting a bug, please include:

- What you tried to do
- What you expected to happen
- What actually happened
- Steps to reproduce the issue
- Environment details (OS, Packer version, Proxmox version)

## Pull Request Guidelines

- Keep each PR focused on a single change.
- Update documentation if behavior changes.
- Ensure all pre-commit hooks and CI checks pass.
- Be open to feedback during code review.

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
